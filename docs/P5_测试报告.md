## 2. 技术选型（与 00 总计划对照）

| 项 | 总计划方案 | 本阶段实际 | 说明 |

## 6. 代码走查关键点

### 6.1 安全 / 合规

- **强制引用**：System Prompt 第 2 条要求事实性陈述必须标注 `[n]`；评测脚本的 `CITATION_RE` 会自动校验
- **禁止剂量建议**：System Prompt 第 3 条；评测脚本 `DOSAGE_DIRECT_RE` 捕获"数字+mg/毫克/片"并要求同时出现"由医生评估/遵医嘱"等否定措辞
- **禁止确定性诊断**：System Prompt 第 4 条
- **免责声明**：System Prompt 第 6 条；评测脚本 `DISCLAIMER_RE` 校验
- **用户数据隔离**：所有会话接口 + SSE 均走 `get_current_user`；跨用户访问统一映射 404

### 6.2 性能 / 稳定性

- **embedding 短路**：Ingest 按内容 SHA256 指纹短路，重复执行零 API 调用（`test_reingest_same_content_is_skipped` 保证）
- **磁盘缓存**：`embed` 默认开启 diskcache，重复 query 零网络
- **SSE 连接生命周期**：`/ask` 端点自建 `SessionLocal`，不走 request-scoped `get_db`，避免流式过程中 Session 被框架回收
- **前端取消**：`fetch + AbortController`，可随时中止生成

### 6.3 可演进点

- `app/services/rag/retriever.py` 只做向量检索，但接口已预留 `distance` 字段，后续接 RRF / Reranker 仅需扩展 `retrieve` 内部
- `chat_service.answer_stream` 与 API 层解耦，将来切换到非流式或切换 provider，仅改 `llm_client`
