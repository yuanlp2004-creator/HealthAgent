## 1. 课题目标拆解

任务书规定三项核心内容：

1. **电子血压计图片信息识别**（OCR）
2. **血压、心跳等健康信息的管理与预测**（数据管理 + 时序预测）
3. **基于大模型的健康信息推理与医疗建议**（LLM + RAG）

对应三项技术能力培养目标：视觉 OCR、大模型推理与预测、RAG。

## 2. 技术选型（已确认）

| 模块 | 方案 |
| --- | --- |
| 后端 | Python 3.10 + FastAPI + SQLAlchemy + Alembic |
| 数据库 | PostgreSQL（结构化数据） + Chroma（向量库） |
| 前端 | React 18 + Vite + TypeScript + Ant Design + ECharts |
| OCR | 本地HOG+SVM分类器 → LCD直达通义千问VL / 非LCD走百度云通用OCR+X坐标聚类；PaddleOCR保留离线可选 |
| 大模型 | 在线 API 通义千问（qwen-plus / qwen-max，主）；DeepSeek 备用 |
| Embedding | 在线 API（text-embedding-v3 或 bge-m3 API） |
| RAG 框架 | LangChain / LlamaIndex（择一） |
| 预测模型 | Prophet / LSTM / ARIMA（对比选型） |
| 用户体系 | JWT + bcrypt，多用户隔离 |
| 部署 | Docker Compose（后端 + 前端 + PG + Chroma） |

## 5. 风险与缓解

| 风险 | 影响 | 缓解策略 |
| --- | --- | --- |
| PaddleOCR/百度云OCR对七段数码管LCD识别精度不足 | OCR模块失败 | LCD分类后直达VLM，跳过OCR；标准字体走OCR主通路 |
| 在线大模型 API 费用/限流 | 问诊不可用 | 缓存高频问答 + 支持切换 provider |
| 医学建议合规风险 | 论文/答辩争议 | 界面显著声明"仅供参考、非诊断" |
| 时序数据样本少导致预测差 | 预测模块弱 | 结合滑动平均与简单规则提示，不过度依赖模型 |
| 截止日期临近 | 延期 | P6/P7 并行、非核心需求（如社交）砍掉 |
