## 2. 关键改动

### 2.1 设计系统

- **主题**：[theme.ts](../frontend/src/theme.ts) 抽出独立 `ThemeConfig`，统一 primary / success / warning / error 四色与圆角/字号；`Layout.headerBg = "#fff"` 使顶栏变亮色，`Card.borderRadiusLG = 12` 让所有卡片视觉柔化
- **间距/字号常量**：`spacing = {xs:4, sm:8, md:16, lg:24, xl:32}`、`fontSize = {xs:12, sm:14, md:16, lg:20, xl:24}`，供未来按 token 对齐
- **公共组件**：
  - `PageHeader`：title + subtitle + extra，`flexWrap: wrap` 在窄屏优雅换行
  - `EmptyState`：薄封装 AntD `Empty`，统一 padding=32，后续可在此集中换图/文案
  - `ErrorBoundary`：`getDerivedStateFromError` + `componentDidCatch`，兜底渲染异常

### 2.2 导航与信息架构

- **由顶部水平 Menu 改为 `Layout.Sider`（可折叠侧栏）**：
  - 5 个菜单项均配 `@ant-design/icons` 图标（DashboardOutlined / CameraOutlined / DatabaseOutlined / MessageOutlined / UserOutlined）
  - 标签按 06 文档重命名：**首页看板 / 拍照录入 / 健康数据 / 智能问诊 / 个人中心**
  - `breakpoint="lg"` + `onBreakpoint`：窄屏自动折叠
- **顶栏**：白色 Header + 左侧折叠按钮 + `Breadcrumb(HealthAgent › 当前页)` + 右侧头像 `Dropdown`（跳转个人中心 / 退出登录）
- **动态 `document.title`**：`App.tsx` 新增 `DocumentTitle` 组件，`useLocation` 监听路由匹配 `ROUTE_META` 后设置 `document.title = "<页面名> · HealthAgent"`
- **登录 redirect 保留**：[RequireAuth](../frontend/src/components/RequireAuth.tsx) 原有 `?redirect=` 机制不变；[LoginPage](../frontend/src/pages/LoginPage.tsx) 登录成功后 `decodeURIComponent(redirect)` 回跳

### 2.3 关键页面打磨亮点

- **登录/注册**：`linear-gradient(135deg, #e6f4ff 0%, #f5f7fa 100%)` 背景 + `HeartFilled` 品牌 Logo + `size="large"` 输入框 + prefix 图标（UserOutlined / LockOutlined / MailOutlined）
- **Dashboard 问候语**：`greeting()` 函数按小时返回 早/中/下/晚/夜深，拼 `user?.nickname || user?.username`
- **ChatPage 布局溢出修复（次要 Bug Fix）**：
  - 主对话 Card 添加 `minWidth: 0`：Flex 项默认 `min-width: auto`，长文本会把容器"撑破"。`minWidth: 0` 允许 Flex 项缩小到内容以下
  - `bodyStyle.minHeight: 0` + 滚动区 `minHeight: 0`：嵌套 Flex 容器里内层 `overflow: auto` 只在父级 `minHeight/width: 0` 时生效
  - 两种气泡都加 `wordBreak: 'break-word'` + `overflowWrap: 'break-word'`，用户气泡 `maxWidth 80%→75%`
  - 侧栏 Card 加 `flexShrink: 0`，避免长消息到来时侧栏被挤压
  - 容器高度 `calc(100vh - 260px)` → `calc(100vh - 220px)` + `minHeight: 480`，适配新的 Sider/PageHeader 垂直占位

## 4. 代码走查关键点

### 4.1 可靠性 / 可维护性

- **主题与路由元数据集中**：所有页面 title/色值改动只需改 [theme.ts](../frontend/src/theme.ts)，避免散落在各页面
- **PageHeader 统一入口**：后续需要加面包屑二级、返回按钮等，只改一个文件
- **`RequireAuth` 未动**：redirect 机制沿用 P2，P6 没有引入回归风险
- **Token 刷新/登录拦截**：[client.ts](../frontend/src/api/client.ts) 未改，保证 P2 P3 既有行为

### 4.2 性能 / UX

- Sider 折叠由 AntD 原生 `Layout.Sider` 实现，不触发重布局抖动
- `document.title` 只在 `location.pathname` 变化时更新，避免同路由下无效写入
- ChatPage 布局改为 `minWidth: 0`/`minHeight: 0` 约束后，长文本/多引用场景不再影响侧栏和输入栏

### 4.3 可演进点

- `spacing` / `fontSize` 常量仅对外导出，暂未在组件中强制引用；下阶段可在新组件中按 token 对齐，逐步替换硬编码数字
- `ROUTE_META` 只存 `title`，后续如要做面包屑多级、页面权限位，可扩展为 `{ title, parent, requireRole }`

## 6. 已知问题 / 后续建议

1. **移动端未适配**（明确不做）：未设置 `<768px` 断点切换单列布局；移动 Safari / 微信内嵌等场景可能出现侧栏遮挡；答辩仅演示桌面 Chrome/Edge。若答辩评委要求移动端，需补 2.4 的三项（断点、粘性按钮、图表可横向滑动）
2. **无 i18n 文件**（明确不做）：中文硬编码在组件中；如果需要英文答辩版本，需要加 `locales/zh-CN.ts` 并抽取文案
3. **无 Lighthouse 评测**（明确不做）：可访问性分数 ≥85 的验收未执行；若被要求可跑 `npm run build && npx lighthouse http://localhost:5173`
4. **无 demo 脚本 / seed 账号**（明确不做）：演示路径未固化为 `docs/demo_script.md`；`yuanlp2004` 账号为真实数据，答辩前建议新建两个 seed 账号（空数据 + 丰富数据）
5. **ChatPage 缺页面级 Vitest**（P5 遗留）：当前仅做手工 UI 验证；P7 视时间盒决定是否补齐
6. **Loading 未完全标准化**：各页面仍独立使用 `Spin`，没有统一的骨架屏组件；可演进点，不阻塞验收
7. **Sider 折叠状态未持久化**：刷新页面后恢复默认展开；可用 `localStorage` 记住用户选择（1 行改动，但非必需）
