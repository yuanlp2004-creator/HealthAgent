## 1. 测试范围

本阶段只交付后端骨架与用户体系，测试聚焦于以下模块：

| 模块 | 覆盖点 |
| --- | --- |
| `app.core.security` | 密码哈希/校验、JWT 生成/解码 |
| `app.api.v1.auth` | 注册、登录、刷新 token |
| `app.api.v1.users` | 当前用户信息、修改资料、修改密码 |
| `app.api.deps` | JWT 鉴权依赖 |
| `app.main` | 应用装配与健康检查 |

OCR、健康数据、RAG 等模块在后续阶段落地，本报告不涉及。

## 2. 测试方法

- **工具**：`pytest` + `pytest-asyncio` + `httpx.TestClient` + `pytest-cov`
- **数据隔离**：每个测试使用独立的内存 SQLite（`StaticPool`），通过 `app.dependency_overrides[get_db]` 注入测试会话，避免文件污染
- **运行命令**：

  ```bash
  cd backend
  python -m pytest -v --cov=app --cov-report=term --junitxml=test-results.xml
  ```

- **结果归档**：`backend/test-results.xml`（JUnit XML）

## 6. 缺陷与处理

| # | 现象 | 根因 | 处理 |
| --- | --- | --- | --- |
| D-01 | 首次运行所有 bcrypt 相关用例失败，报 `password cannot be longer than 72 bytes` | `passlib 1.7.4` 在探测 bcrypt 版本时构造了 >72 字节字符串，新版 `bcrypt 4.1+` 对此做了硬校验 | 锁定 `bcrypt==4.0.1`，写入 `pyproject.toml` 与 `README.md` |

## 7. 风险与后续

- **Postgres 路径未在本阶段实测**：CI 环境使用 SQLite，Docker 部署的 Postgres 链路将在 P2 末尾并入 Docker 启动冒烟测试。
- **速率限制/锁定未实现**：登录失败锁定、暴力破解防护计划在 P7 性能/安全测试阶段补充。
- **Alembic 迁移尚未启用**：当前用 `Base.metadata.create_all` 建表，切换到 Postgres 前需在 P2 初补 Alembic revision。
