# P6 前端集成与 UI 打磨

> 周期：2026-04-21 ~ 2026-05-05（2 周）
> 目标：整合前序阶段各自页面，做视觉/交互统一与移动端适配，为答辩演示呈现完整体验。

## 1. 目标

- 全站视觉统一、响应式可用（桌面 + 移动浏览器）
- 全链路跑通：注册 → 拍照 → 识别 → 入库 → 看板 → 问诊
- 演示路径稳定，无明显 bug

## 2. 任务拆解

### 2.1 设计系统收敛（第 1 周前半）

- 统一主题：AntD `ConfigProvider` 设置主色、圆角、字体
- 统一间距（4/8/16/24）、字号（12/14/16/20/24）
- 统一空状态、错误页、Loading 骨架屏
- 图标统一使用 `@ant-design/icons`

### 2.2 信息架构与导航（第 1 周前半）

- 顶部/侧栏主导航：**首页看板 / 拍照录入 / 健康数据 / 智能问诊 / 个人中心**
- 面包屑与页面标题对齐路由
- 未登录跳转保留 `redirect` 参数，登录后回跳

### 2.3 关键页面打磨（第 1 周）

| 页面 | 打磨点 |
| --- | --- |
| 登录/注册 | 背景插画、表单校验提示、自动聚焦 |
| 首页看板 | 顶部问候语（按时间/用户名）、核心卡片、图表、异常提醒 |
| 拍照录入 | 拖拽/相机/相册三种入口；识别中骨架屏；低置信度红框提示 |
| 健康数据 | 时间筛选器、分页、导出、批量删除 |
| 智能问诊 | 流式 typing 光标、引用 Popover、快捷问题胶囊 |
| 个人中心 | 阈值设置、修改密码、注销（软删） |

### 2.4 移动端适配（第 1 周尾）

- 全站断点：`<768px` 切换单列布局
- 关键操作（拍照、问诊输入）底部粘性按钮
- 首页图表改为可横向滑动

### 2.5 细节与可用性（第 2 周前半）

- 全站错误边界 `ErrorBoundary`
- 全站 Loading / Empty / Error 三态标准化
- 键盘可达性：表单回车提交、Tab 顺序正确
- SEO/Title：根据路由动态设置 `<title>`
- i18n 预留（本阶段只做中文；文案走 `locales/zh-CN.ts`）

### 2.6 端到端联调与演示脚本（第 2 周后半）

- 列出答辩演示的 5 分钟路径，按路径走查
- 准备 2 个种子账号（空数据 / 丰富数据）
- 录制 3 分钟演示视频备用

## 3. 交付物

- 打磨后的 `frontend/`
- 演示脚本（`docs/demo_script.md`）
- 2 个预置种子账号与数据 seed 脚本

## 4. 验收标准

- 关键页面在 Chrome / Edge / 移动端 Safari 均正常
- Lighthouse 可访问性评分 ≥ 85
- 演示路径 3 次执行均无报错

## 5. 风险

| 风险 | 缓解 |
| --- | --- |
| 移动端样式碎片化 | 只保证主流浏览器；老旧内核在论文中说明不支持 |
| 演示时网络/API 抖动 | 准备离线录制视频作为 Plan B |
