## 1. 测试范围

本阶段交付 Web 前端骨架并打通 P1 后端接口，测试聚焦以下内容：

| 模块 | 覆盖点 |
| --- | --- |
| `frontend/src/store/auth` | zustand auth store（登录/注册/登出/bootstrap） |
| `frontend/src/utils/error` | 统一错误信息提取（字符串 detail / pydantic 列表 / 状态码回落） |

- **工具**：`vitest 2.1.9` + `jsdom` + `@testing-library/react` + `@testing-library/jest-dom`

- **脚本**：使用 `httpx.Client` 在真实 HTTP 栈上依次调用各接口，对状态码与 payload 做断言，结束打印 `PASS/FAIL` 汇总。

## 6. 缺陷与处理

| # | 现象 | 根因 | 处理 |
| --- | --- | --- | --- |
| D-02 | `npm install` 报 `'node' 不是内部或外部命令`，esbuild 安装失败 | winget 新装的 Node 未刷新到当前 Git Bash 会话 PATH | 在命令前临时导出 `PATH="/c/Program Files/nodejs:$PATH"` |
| D-03 | 首次 `npm run build` 使用 `tsc -b` 将 `.ts/.tsx` 编译成 `.js` 写回 `src/`，导致 vitest 重复收集 | `tsc -b` 默认启用 emit | `tsconfig.json` 增加 `"noEmit": true`；`build` 脚本改为 `tsc --noEmit && vite build` |
| D-04 | Docker Desktop 安装包下载两次 `InternetOpenUrl() failed 0x80072ee2` | 安装镜像 ~800 MB 网络不稳 | 放弃本机容器冒烟，用 uvicorn+SQLite 做端到端冒烟；Docker/Postgres 验证推迟到 P7 |

## 7. 风险与后续

- **未做容器内冒烟**：`docker-compose.yml` 配置已经就绪（Postgres 16 + FastAPI），但本机 Docker Desktop 暂时装不上，须在 P7 部署阶段补一次 `docker compose up` 冒烟（包含 Postgres 真实迁移）。
- **bundle 体积偏大**：AntD 全量导入导致 gzip 280 KB，计划在 P6 启用按需引入或 `manualChunks`。
- **前端尚未接入 E2E（Playwright）**：当前通过"vitest 单测 + 后端 httpx 冒烟"两层替代，最终答辩前在 P7 考虑补一轮 Playwright 录屏演示脚本。
- **Alembic 迁移仍未启用**：SQLite/Postgres 当前都走 `Base.metadata.create_all`，在接入 OCR/健康数据表（P3/P4）前必须切换到 Alembic 以便后续演进。
