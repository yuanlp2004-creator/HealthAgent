### 3.2 后端骨架（第 1 周）

- 依赖：`fastapi`, `uvicorn[standard]`, `sqlalchemy`, `alembic`, `psycopg[binary]`, `pydantic-settings`, `passlib[bcrypt]`, `python-jose[cryptography]`, `python-multipart`
- 配置：`core/config.py` 使用 `pydantic-settings` 读取 `.env`
- 统一响应结构：`{code, message, data}`；统一异常处理器
- 日志：`loguru`，请求中间件注入 `request_id`
- 健康检查：`GET /healthz`

### 3.3 数据库与迁移（第 1 周尾）

- `alembic init`，基于 `models/` 自动生成迁移
- 创建 `users` 表（见 P1 设计），预留其他表的空模型文件

### 3.4 用户体系（第 2 周）

**接口：**

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | /api/v1/auth/register | 注册，返回用户 + token |
| POST | /api/v1/auth/login | 登录，返回 token |
| POST | /api/v1/auth/refresh | 刷新 token |
| GET  | /api/v1/users/me | 当前用户信息 |
| PATCH| /api/v1/users/me | 修改资料（昵称、性别、生日等） |
| POST | /api/v1/users/me/password | 改密码 |

**安全：**

- 密码 bcrypt 哈希（`passlib`），不明文存储
- JWT：access 15min + refresh 7d；`HS256` + 强随机 secret
- 登录失败 5 次锁定 10 分钟（简单内存计数即可）
- CORS 仅允许前端源

### 3.5 前端骨架（第 2 周并行）

- Vite + React 18 + TS + AntD 5 + React Router 6 + Zustand
- `src/api/http.ts`：axios 拦截器，自动附带 token、处理 401 刷新
- 路由守卫：未登录跳转 `/login`
- 页面：登录、注册、个人中心（空壳，后续阶段填充）

### 3.6 容器化与 CI（第 3 周）

- `docker-compose.yml`：`backend`, `frontend`, `postgres`, `chroma`
- 后端 Dockerfile 多阶段构建；前端 Dockerfile nginx 部署
- GitHub Actions（若用 GitHub）：lint + pytest + 前端 build

### 3.7 联调与收尾（第 3.5 周）

- 编写 README 与 `.env.example`

## 4. 交付物

- 可运行的 `backend/` 与 `frontend/` 代码

## 5. 验收标准

- 克隆仓库 → `cp .env.example .env` → `docker-compose up -d` 全栈启动无报错
- 前端能完成注册 → 登录 → 查看个人信息 → 退出
- 未带 token 调用受保护接口返回 401
- pytest 全绿

## 6. 风险

| 风险 | 缓解 |
| --- | --- |
| JWT 密钥泄露 | `.env` 不入库，`JWT_SECRET` 生成脚本 |
| bcrypt 在 Windows 下编译问题 | 文档注明使用 Docker 或 `passlib[bcrypt]` 预编译 wheel |
| 前后端 CORS 报错 | 提供本地代理配置示例 |
