# P4 健康数据管理与预测

> 周期：2026-02-21 ~ 2026-03-20（4 周）
> 目标：围绕 `health_records` 表建立完整的 CRUD、可视化看板、统计聚合与趋势预测，覆盖课题第二项内容。

## 1. 目标

- 用户可查看 / 编辑 / 删除 / 导出历史健康数据
- 提供日 / 周 / 月聚合图表（ECharts）
- 提供未来 7 天趋势预测与异常阈值告警

## 2. 输入

- P3 产生的 `health_records`（由 OCR 或手动录入）
- 用户基础信息（性别、年龄，用于个性化阈值）

## 3. 任务拆解

### 3.1 数据模型补全（第 1 周前半）

- 在 `health_records` 增加索引：`(user_id, record_time desc)`
- 新增字段（若缺）：`temperature`, `weight`, `spo2`，允许空，方便扩展
- Alembic 迁移脚本

### 3.2 CRUD 接口（第 1 周）

```
POST   /api/v1/health/records
GET    /api/v1/health/records?from&to&type&page&size
GET    /api/v1/health/records/{id}
PATCH  /api/v1/health/records/{id}
DELETE /api/v1/health/records/{id}
GET    /api/v1/health/records/export.csv
```

- 严格按 `user_id` 隔离查询
- 分页采用 cursor 或 offset（数据量不大，用 offset 即可）

### 3.3 统计聚合（第 2 周前半）

```
GET /api/v1/health/stats?metric=bp&granularity=day&from&to
  -> [{date, avg_systolic, avg_diastolic, max, min, count}, ...]
```

- 后端用 SQL `date_trunc` 做聚合，避免前端大量计算
- 前端 ECharts 绘制折线图 + 面积图，支持 1 周 / 1 月 / 3 月切换

### 3.4 异常阈值与告警（第 2 周后半）

- 内置阈值（可被用户覆盖）：
  - 高血压：收缩压 ≥ 140 或 舒张压 ≥ 90
  - 低血压：收缩压 < 90 或 舒张压 < 60
  - 心率异常：< 50 或 > 100（静息）
- 每条记录入库后计算 `flags`，列表页用颜色标识
- 首页看板卡片展示"近 7 天异常次数"

### 3.5 趋势预测（第 3 周）

**方案对比与选型：**

| 方案 | 优点 | 缺点 |
| --- | --- | --- |
| 滑动平均 + 线性外推 | 无依赖、快 | 精度有限，但可做基线 |
| Prophet | 处理缺失、节律性好 | 依赖 pystan |
| LSTM | 可捕捉长期依赖 | 数据量不足、训练成本高 |

**推荐**：以 Prophet 为主，滑动平均作为回退。当用户记录少于 14 条时自动退化到回退方案。

**接口：**

```
GET /api/v1/health/forecast?metric=systolic&horizon=7
  -> {
       history: [...],
       forecast: [{date, yhat, yhat_lower, yhat_upper}, ...],
       method: "prophet" | "fallback",
       note: "..."
     }
```

- 预测在后端同步计算（数据量小）；如超 500ms 再考虑缓存

### 3.6 前端看板（第 4 周）

- 首页 `/dashboard`：
  - 顶部卡片：最新一次测量、7 天均值、异常次数
  - 折线图：收缩压/舒张压/心率三线联动
  - 预测视图：虚线 + 置信区间阴影
  - 右侧：异常提醒列表
- 历史页 `/records`：表格 + 筛选 + 导出 CSV

## 4. 交付物

- 健康数据 CRUD、统计、预测后端接口
- 前端 Dashboard 与 Records 页面
- Prophet 预测服务封装 + 单测
- 阈值规则配置（默认值 + 用户覆盖表 `user_thresholds`）

## 5. 验收标准

- 所有列表/统计接口按用户隔离（跨用户访问返回 404/403）
- 预测接口在 <14 条数据时自动回退且不抛错
- 看板在 1000 条记录下首屏渲染 ≤ 1s
- 导出 CSV 字段齐全，编码 UTF-8 with BOM（Excel 可直接打开）

## 6. 风险

| 风险 | 缓解 |
| --- | --- |
| Prophet 安装失败（Windows） | 使用 Docker 或退回 `statsforecast` / 滑动平均 |
| 数据稀疏导致预测方差大 | 在 UI 上显示置信区间 + 文字提示"仅供参考" |
| 时区问题 | 后端统一存 UTC，前端按用户时区显示 |
