# P7 测试、部署与毕业论文

> 周期：2026-05-06 ~ 2026-05-20（2 周）
> 目标：完成系统测试、部署上线、毕业论文与答辩 PPT，达到可提交可答辩状态。

## 1. 目标

- 通过系统测试并形成测试报告
- 系统部署到可公网或校内网访问的服务器
- 毕业论文定稿（含摘要、相关工作、系统设计、实现、评测、结论）
- 答辩 PPT + 演示视频

## 2. 任务拆解

### 2.1 测试（第 1 周前半）

**单元测试**（后端 pytest，目标覆盖率 ≥ 70%）：
- auth / ocr 字段抽取 / 健康数据 CRUD / 预测服务 / RAG 检索

**集成测试**：
- 用 `pytest + httpx.AsyncClient` 跑完整链路
- 关键场景：注册→登录→OCR→入库→看板→问诊

**性能测试**（`locust` 或简单脚本）：
- OCR 并发 10 QPS 下 P95 ≤ 5s
- 问诊 SSE 首 token P95 ≤ 4s

**用户测试**：
- 邀请 3~5 位同学实际使用，收集可用性问题

**测试报告** `docs/test_report.md`：包含用例、通过率、bug 清单与修复记录。

### 2.2 部署（第 1 周后半）

- 部署目标：校内 Linux 服务器 / 云服务器（1 台 2C4G 足够）
- 方式：`docker-compose -f docker-compose.prod.yml up -d`
- HTTPS：Caddy 自动申请 Let's Encrypt 证书
- 备份：PG 每日 `pg_dump` 到服务器本地 + 手动拷贝
- 监控：简单 `uptime-kuma`（可选）
- 域名：使用免费二级域名或教师提供

### 2.3 毕业论文（全程滚动，本阶段定稿）

建议章节：

1. 绪论（背景、意义、国内外现状、主要工作、章节安排）
2. 相关技术（OCR / LLM / RAG / 时序预测）
3. 需求分析与系统设计
4. 关键模块实现
   - 血压计图像识别（算法 + 评测）
   - 健康数据管理与预测（模型对比）
   - RAG 与大模型问诊（检索策略 + prompt 工程）
5. 系统测试与评测
6. 结论与展望

写作要点：
- 每章配图（架构、ER、流程、界面截图、实验曲线）
- 评测章节要有对比实验与数据表格
- 参考文献 ≥ 15 篇（含任务书指定 2 篇）

### 2.4 答辩准备（第 2 周）

- PPT 结构：背景（1）→ 系统总览（1）→ 三大模块各 2 页 → 评测（2）→ 创新与不足（1）→ Q&A（1）
- 演示视频 3 分钟，展示核心路径
- 预演 ≥ 2 轮，记录常见问题并准备答案

## 5. 风险

| 风险 | 缓解 |
| --- | --- |
| 临近截止日系统被改坏 | 从 `v0.9` 起只接受 bugfix；重构冻结 |
| 答辩当天网络故障 | 演示视频 + 本地离线环境 Plan B |
| 论文查重不过 | 提前一周查重并修改 |
| 部署服务器挂掉 | 保留本地 `docker-compose up` 的应急方案 |
